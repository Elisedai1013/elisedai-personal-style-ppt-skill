# EliseDai Personal Style PPT Skill

用一套稳定的个人视觉语言，把逐字稿、文章、大纲或已有 PPT，转成文字主导、黑白鲜曜蓝、适合讲述的 AI-native 演示文稿。

这不是一套反复套用的页面模板。它固定的是判断：一页只讲一个观点，蓝色承担意义，截图保留证据，配图负责推进叙事。

![AI Builders 解读 02 封面](examples/episode-02/cover.jpg)

## 这套风格解决什么问题

很多 AI 生成的 PPT 看起来“完整”，但容易出现几个问题：每页像卡片拼装、颜色越来越多、截图只是被塞进矩形框、插图与观点没有关系、文字层级太多，最后更像一份自动排版的报告，而不是一场可以讲出来的演示。

EliseDai Personal Style PPT Skill 提供的是另一种做法：

- **先讲清楚，再设计。** 每页只有一个可说出口的判断。
- **固定视觉语言。** 只使用黑、白、灰和鲜曜蓝 `#276EF1`。
- **用三种画布控制节奏。** 白场解释，黑场制造张力，蓝场强调转折与结论。
- **截图是证据，不是装饰。** 优先保证完整、可读和不失真，再考虑统一风格。
- **配图不是固定简笔画。** 根据内容选择手绘隐喻、几何舞台、全场景插图、照片或截图。
- **保留可编辑性。** 文字、简单结构、图表与页面骨架继续留在 PPT 中编辑。

## 真实案例：AI Builders 解读 02

案例主题：**AI 能生成十个方案以后，Builder 为什么要删掉九个？**

输入不是一份已经排好的页面，而是一段围绕舞台导演、AI 产品构建和选择判断展开的解读逐字稿，其中还混合了演讲截图、现场画面和抽象观点。

这套 Skill 先把整场演示收敛成一条主线：

> AI 可以更快地产生选项，但 Builder 的价值，是找到作品的主心骨，决定最后留下什么。

然后再为不同内容选择不同页面形式。

### 1. 蓝场负责建立主题，不堆信息

封面只留下核心问题、讲者信息和一个舞台隐喻。鲜曜蓝不是装饰色，而是整套演示中“选择与行动”的信号。

![第二期封面](examples/episode-02/cover.jpg)

### 2. 白场承载证据，让截图融入页面

当两张截图需要比较时，不把它们塞进不同颜色的卡片。保持原始画面比例和可读区域，用一致的尺寸、留白、标题与注释建立关系。

![同一份剧本，可以是完全不同的演出](examples/episode-02/screenshot-comparison.jpg)

### 3. 抽象观点转换成视觉隐喻

“先给作品一条主心骨”没有被做成流程图，而是转化为一条贯穿材料、把零散选择组织起来的蓝色脊柱。

![先给作品一条主心骨](examples/episode-02/concept-spine.jpg)

“观众不是最后验收的人”则使用人物、空间和一条蓝色路径，表达观众如何进入作品，而不是再加一组解释卡片。

![观众是作品的一部分](examples/episode-02/audience-line-art.jpg)

### 4. 黑场为截图提供舞台，而不是强制改色

深色页面中的产品截图保持原有界面颜色。统一感来自裁切、比例、位置、标题和舞台关系，而不是把所有截图都染成蓝色。

![一个双击 Command 排练了一年](examples/episode-02/screenshot-dark-stage.jpg)

### 5. 结论页把判断变成一个动作

“删掉九个”没有被解释成九条原则，而是做成一排被淘汰的方案与唯一被保留的蓝色出口。视觉本身承担了结论。

![删掉九个，并为留下的那个负责](examples/episode-02/metaphor-delete-nine.jpg)

最后回到舞台：AI 可以加快写剧本，Builder 决定最后的演出。开头提出的问题在结尾被真正回答。

![AI 可以更快地写出剧本，Builder 决定最后的演出](examples/episode-02/closing.jpg)

## 哪些东西固定，哪些东西可以变化

| 固定的个人语言 | 根据内容变化的部分 |
| --- | --- |
| 黑、白、灰、鲜曜蓝 | 页面使用白场、黑场还是蓝场 |
| 腾讯体优先的排版层级 | 截图、照片、手绘、几何或全场景插图 |
| 一页一个观点 | 左右、全幅、窗口、重叠或留白构图 |
| 蓝色只承担一个语义角色 | 蓝色可以是路径、光、行动、结果或整页场域 |
| 截图不失真、不误裁 | 依据叙事选择原色、轻度清理或局部处理 |
| 统一的边距、标题基线和页码 | 每页的视觉轮廓与节奏 |

## 截图与图片怎么处理

Skill 会先判断图片在这一页的角色：

- **Evidence：** 截图、图表、界面或现场照片。优先保持原色、比例和可读信息。
- **Context：** 用来交代人物、场景或来源。可以通过裁切和留白融入页面。
- **Metaphor：** 用来表达抽象概念。应根据当前观点生成，而不是复用通用图标。
- **Atmosphere：** 只负责情绪和空间时，可以使用灰度、浅蓝或双色处理。

默认不把截图放在彩色底板上，也不为了“统一”而强制全蓝。真正的统一来自：相同角色的图片拥有一致的可见尺寸、裁切逻辑、圆角、间距和标题关系。

## 工作流程

1. 定义观众最后应该理解什么。
2. 把内容收敛成一条累计推进的叙事主线。
3. 为每页写一句可以直接说出口的观点标题。
4. 选择白场、黑场或鲜曜蓝场。
5. 决定这一页需要证据、隐喻、场景还是纯文字。
6. 生成或处理图片，保持截图完整性。
7. 构建可编辑 PPT，并运行建议型视觉 Hook。
8. 渲染全部页面，逐页检查文字、截图、字体、重叠和节奏。

Hook 不会代替设计判断，也不会强制阻断交付。它负责提醒容易忽略的问题，例如截图太小、裁切失真、连续多页只使用截图、蓝色沦为装饰、图片尺寸不一致或字体发生替换。

## 安装

仓库中的 Skill 位于：

```text
skills/elisedai-personal-style-ppt/
```

将这个目录复制到你的 Codex Skills 目录：

```bash
cp -R skills/elisedai-personal-style-ppt ~/.codex/skills/
```

也可以通过支持 GitHub Skill 导入的平台安装本仓库。

## 怎么使用

从内容生成新 PPT：

```text
使用 $elisedai-personal-style-ppt，把这份逐字稿做成一套 16:9 的讲述型 PPT。
```

优化已有 PPT：

```text
使用 $elisedai-personal-style-ppt，保留原有内容和截图，把这份 PPT 改成 EliseDai 个人风格。
```

强调截图完整性：

```text
使用 $elisedai-personal-style-ppt 优化这份 PPT。截图不要强制改色，先保证完整、可读和不突兀。
```

## 字体说明

视觉系统优先使用：

- 标题与关键判断：`TencentSans W7`
- 正文、英文、数字和说明：`TencentSans W3`

腾讯字体文件没有包含在公开仓库中。请使用已经获得授权的本地字体；如果运行环境没有腾讯体，Skill 会建议使用 Source Han Sans SC 或 Noto Sans CJK SC，并在交付时说明字体替换。

## 设计原则

好的个人风格，不是让每一页长得一样。

它应该让不同主题、不同截图和不同配图进入同一套判断系统：文字始终清楚，颜色始终克制，图片始终有角色，蓝色始终有意义。
